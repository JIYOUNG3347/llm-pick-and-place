명령 한 줄이
관절 명령이 되기까지

흰 카드는 처리(노드), 그 사이 회색 패킷은 이동하는 값입니다. 모든 모듈은 서로의 내부를 모르고 오직 5개의 타입 계약으로만 대화하며, 각 노드는 받은 값을 다음 노드가 먹을 타입으로 변환할 뿐이에요.

노드 역할

입출력 · 환경 인식 Perception 계획 Planner 실행 Executor

데이터 계약 (클릭 시 흐름 추적)

↳ 타입을 누르면 어디서 생성돼 어디로 들어가는지 전체가 표시됩니다.

입력
사용자 명령

자연어 명령 한 줄 (한국어/영어).

전달
예: "파란 박스를 빨간 박스 위에 올려놔"
환경 · 초기화
env.reset()

씬을 리셋하고 시뮬레이터의 날것 관측을 돌려줍니다.

↺ 실패 시 perceive·plan으로 복귀
전달 · 날것 관측
├ gt_objects : [{ label, pose }] └ robot : 관절·EE 상태 (raw)
인식 · Perception
OraclePerception.perceive(obs)

GT 포즈를 신뢰해 의미 있는 씬 스냅샷으로 변환 (confidence 1.0, 카메라 인식 없음).

전달
├ t : int ├ objects : list[ObjectState] │ └ { id, label, pose{ position[3], quaternion[4] }, confidence, bbox } └ robot : { joint_positions[n], ee_pose, gripper:float, holding }
예: objects=[red_cube, blue_cube] · robot.holding=None
계획 · LLM 에이전트
LlmPlanner.plan(world, instruction)

도구 호출 루프로 명령을 기술 호출 시퀀스로 변환. 실패 시 위치-프롬프트 JSON → rule_based 폴백.

전달
└ steps : list[] SkillCall{ skill:str, args:dict }
예: pick{ label:"blue_cube" } → place{ label:"blue_cube", target:"red_cube" }
실행 · DiffIK
SkillExecutor.run(call, world)

SkillCall을 하나씩 상태머신으로 실행 (pick: APPROACH→DESCEND→CLOSE→LIFT). 진짜 일하는 안쪽 루프 ↓

매 틱 반복 · SUCCESS/FAILURE까지
skill.act() → → env.step(Action) → perceive() ↻
Action ├ joint_targets : ndarray[n] ├ gripper : float └ ee_target : Pose | None 예: ee_target = blue_cube + tcp_offset_z(0.107) · gripper = close
전달
├ status : RUNNING | SUCCESS | FAILURE ├ message : str └ world_after :
예: status=SUCCESS · message="picked blue_cube"
판정 · Orchestrator
성공 여부 + 재계획 예산 검사

모든 SkillCall 성공 → 종료. 하나라도 실패하고 예산이 남으면 갱신된 WorldState로 다시 perceive·plan.

✓ 전부 성공
✗ 실패 · 예산 남음
↺ 위로 복귀
5 contracts · instruction → WorldState → Plan → Action → ActionResult
